## 🐛 Bug Fixed This Session

### `IntelligenceLayer.analyze()` — `..take(maxVisible)` wasn't truncating list

**File**: `lib/core/domain/services/intelligence_layer.dart`

Code used cascade `..take(maxVisible)` which calls `take()` but discards the return value — the list was never truncated to 3 insights.

```dart
// ❌ BEFORE (buggy) — cascade discards the return value of take()
return raw
    .where(...)
    .toList()
  ..sort(_byPriority)
  ..take(maxVisible);  // ← return value ignored!

// ✅ AFTER (fixed)
final filtered = raw
    .where(...)
    .toList()
  ..sort(_byPriority);
return filtered.take(maxVisible).toList();
```

**Impact**: The `InsightsPanel` was potentially receiving N insights (unlimited), saturating the UI in cases where many rules fired simultaneously. The 3-insight limit now works correctly.

## ✅ What was migrated (completed 2026-05-08)

### 1. `health_screen.dart` → migrated to `financialSnapshotProvider`

**Was**: 4 providers + manual `FinancialCalculatorService.calculateHealthScore()` call.
**Now**: `financialSnapshotProvider` — score and sub-scores from snapshot directly.

```dart
// ✅ AFTER
final snap = ref.watch(financialSnapshotProvider);
final score = snap.healthScore; // consistent with dashboard HealthGaugeCard
```

### 2. `expense_breakdown.dart` → migrated to `categoriesRefProvider`

**Was**: `categoriesMapProvider` → `Map<String, Category>`
**Now**: inline map from `categoriesRefProvider` → `Map<String, CategoryRef>`

```dart
// ✅ AFTER
final catsRef = { for (final c in ref.watch(categoriesRefProvider)) c.slug: c };
```

### 3. `quick_add_bottom_sheet.dart` + `edit_expense_bottom_sheet.dart` → migrated to `categoriesRefProvider`

**Was**: `categoriesStreamProvider` → `AsyncValue<List<Category>>`
**Now**: `categoriesRefProvider` → `List<CategoryRef>` (synchronous, never null)

Also fixed: `_subcategories` map keys were UPPERCASE (`'HOUSING'`) but slugs are lowercase (`'housing'`) — lookup never matched. Fixed to lowercase.

### 4. `transactions_screen.dart` — `isFixed` / `deleteFixedSeriesFrom` pattern retired

**Was**: Swipe delete on isFixed expense → "Delete entire series" dialog calling `deleteFixedSeriesFrom()`
**Now**: Legacy warning dialog (Scenario B) — single delete only, with note explaining conversion to RecurringRule

`deleteFixedSeriesFrom` marked `@Deprecated` in `expense_repository.dart`.
